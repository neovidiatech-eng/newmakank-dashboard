# عروض الـ Bundle ("اشتري X واحصل على Y مجانًا") — داشبورد المتجر/الأدمن

> آخر فيتشر اتأكد منها حي: العرض بيدعم إن العميل يختار مقاسات مختلفة لكل قطعة (مدفوعة أو هدية) — الشاشة دي بتتحكم في **القواعد** اللي بتحدد إيه المسموح والممنوع وقت ما العميل يطلب من العرض ده على الموبايل.

## إنشاء عرض جديد

### `POST /bundles` (multipart — فيه صورة)
**Auth:** Store (متجره هو) أو Admin

| الحقل | مطلوب | الوصف |
|---|---|---|
| `title`, `description` | ✅ (JSON ثنائي اللغة) | اسم ووصف العرض |
| `image` | ✅ | صورة العرض |
| `storeId` | ✅ | المتجر بتاع العرض |
| `requiredPaidQuantity` | ✅ | عدد القطع "المدفوعة" في العرض (مثال: 2) |
| `freeQuantity` | ✅ | عدد قطع "الهدية" (مثال: 1) |
| `paidServiceIds` و/أو `paidCategoryIds` | لازم واحد منهم | إيه المنتجات/الفئات المسموح تتحسب "مدفوعة" |
| `freeServiceIds` و/أو `freeCategoryIds` | لازم واحد منهم | إيه المنتجات/الفئات المسموح تتحسب "هدية" |
| `isActive` | اختياري (افتراضي: شغال) | تفعيل/تعطيل العرض |
| `startDate` / `endDate` | اختياري | فترة سريان العرض |

### إعدادات المقاس (لو المنتجات في العرض عندها مقاسات)

| الحقل | القيم | الوصف |
|---|---|---|
| `paidSizeRule` | `ANY` (افتراضي) \| `NAME` | `ANY`: العميل يختار أي مقاس لكل قطعة مدفوعة، وممكن يختار مقاسات مختلفة لكل قطعة. `NAME`: كل القطع المدفوعة لازم تكون مقاس معيّن بالاسم (حدده في `paidRequiredSizeName`) |
| `paidRequiredSizeName` | نص (زي "Large") | **إجباري لو `paidSizeRule: NAME`** — بيتقارن بالاسم من غير حساسية لحالة الأحرف، وبيقبل أي لغة (عربي أو إنجليزي) |
| `freeSizeRule` | `ANY` (افتراضي) \| `NAME` | نفس الفكرة، لكن لقطعة الهدية |
| `freeRequiredSizeName` | نص | **إجباري لو `freeSizeRule: NAME`** |

**مثال استخدام:** لو عايز عرض "اشتري ٢ حجم كبير واحصل على ١ حجم صغير هدية" بالتحديد (مش أي مقاس)، تحط:
```json
{ "paidSizeRule": "NAME", "paidRequiredSizeName": "Large", "freeSizeRule": "NAME", "freeRequiredSizeName": "Small" }
```
لو عايز مرونة كاملة (العميل يختار أي مقاس لأي قطعة، زي المثال اللي جرّبناه: صغير + كبير مدفوع، وكبير هدية)، سيب الاتنين على `ANY` (الافتراضي، مش لازم تبعتهم أصلاً).

### إعدادات "أقصى قيمة للهدية" — مهم جدًا تفهمها كويس

بما إن العميل ممكن يختار أي مقاس للهدية (لو `freeSizeRule: ANY`)، محتاجين نتحكم هل نسيبه يختار مقاس غالي كهدية ولا لأ. القاعدة دي بتتحدد بـ:

| `freeValueRule` | المعنى | يحتاج |
|---|---|---|
| `NO_CAP` | مفيش حد أقصى — العميل يقدر يختار أي مقاس/سعر كهدية | — |
| `CAP_TO_CHEAPEST_PAID` **(الافتراضي)** | قيمة الهدية متتعديش سعر **أرخص قطعة مدفوعة** اختارها العميل في نفس الطلب | — |
| `MAX_FREE_VALUE` | فيه سقف سعر ثابت للهدية، بغض النظر عن اللي اتدفع | لازم تبعت `maxFreeItemValue` (رقم) |

**مثال حي جربناه:** عرض "٢ مدفوع + ١ هدية" على منتج له مقاسين (صغير ٦٠ جنيه، كبير ٧٥ جنيه بعد خصم). لو حطينا `freeValueRule: CAP_TO_CHEAPEST_PAID` والعميل اختار (صغير + كبير) كمدفوع، هيتقفله يختار "كبير" كهدية (لأنه أغلى من أرخص قطعة دفعها وهي الصغير) — هيضطر يختار "صغير" بس كهدية. لو حطينا `NO_CAP`، يقدر يختار أي حاجة.

**نصيحة للشاشة:** لما تعمل فورم إنشاء/تعديل عرض، اعرض القيم دي كـ dropdown واضح بشرح مبسط لكل خيار (زي الجدول فوق) — عشان صاحب المتجر يفهم أثرها على تجربة العميل قبل ما يحفظ العرض.

## تعديل عرض

### `PATCH /bundles/:id` (multipart)
**Auth:** Store (عرضه هو بس) أو Admin — نفس حقول الإنشاء، كلها اختيارية.

## عرض العروض

### `GET /bundles?storeId=` (لازم `storeId` أو `id`، مقصود عشان محدش يشوف عروض كل المنصة)
**Auth:** Visitor (مفتوح) — بيرجع نفس شكل الإنشاء بالكامل:

```json
{
  "id": 7, "storeId": 1,
  "title": {...}, "description": {...}, "image": "uploads/...",
  "isActive": true, "type": "BUY_X_GET_Y_FREE",
  "requiredPaidQuantity": 2, "freeQuantity": 1,
  "paidSizeRule": "ANY", "paidRequiredSizeName": null,
  "freeSizeRule": "ANY", "freeRequiredSizeName": null,
  "freeValueRule": "CAP_TO_CHEAPEST_PAID", "maxFreeItemValue": null,
  "startDate": null, "endDate": null,
  "ScopeServices": [ { "role": "PAID", "serviceId": 1, "Service": {...} }, { "role": "FREE", "serviceId": 1, "Service": {...} } ],
  "ScopeCategories": []
}
```

### `DELETE /bundles/:id`
**Auth:** Store (عرضه هو بس) أو Admin

## اتأكدنا منه حي

عملنا عرض حقيقي بقاعدة `paidSizeRule: ANY` و`freeValueRule: NO_CAP`، وطلبنا من الموبايل بمقاسات مختلفة للمدفوع والهدية — السعر اتحسب صح والهدية اتحسبت بصفر جنيه بالظبط. نضّفنا بيانات الاختبار بعدها.
